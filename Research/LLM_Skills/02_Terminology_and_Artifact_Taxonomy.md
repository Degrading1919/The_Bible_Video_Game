## Common category errors

The first common error is **treating ŌĆ£skillŌĆØ as the synonym for any reusable AI artifact**. OpenAIŌĆÖs skills docs and OpenAIŌĆÖs own OSS case study both point in the opposite direction: skills are specifically good for repeatable workflows because they can carry richer instructions, scripts, and references without bloating context up front. If the project uses ŌĆ£skillŌĆØ for policy, identity, role, reference corpus, tool registry, and evaluation all at once, routing becomes ambiguous and maintenance becomes harder. ŅłĆciteŅłéturn5view0Ņłéturn30view0Ņłü

The second error is **putting durable policy only in a skill**. This creates a silent failure mode: if the skill is not triggered, the policy disappears. CodexŌĆÖs own docs instead assign persistent repository guidance to `AGENTS.md`, while the memories docs explicitly warn to keep required team guidance in `AGENTS.md` or checked-in documentation rather than rely on memory. ŅłĆciteŅłéturn13view2Ņłéturn29view0Ņłéturn10view3Ņłéturn16view0Ņłéturn13view0Ņłü

The third error is **confusing role with persona**. The fourth error is **confusing commands, prompts, and skills**. ## Implications for The Bible Video Game

### Findings

For this project, the central terminology decision should be to reserve **skill** for what OpenAI/Codex already appear to mean by it: a reusable package for workflow and operational expertise. ### Recommendations

A practical normalization for this project would be:

- **Role** = human or agentic decision lens and ownership boundary.
- **Persona** = communication style only.
- **Policy** = durable constraint that should apply even if no skill is triggered.
- **Domain reference** = authoritative subject-matter source.
- **Skill** = reusable workflow package.
- **Tool-operation skill** = a skill only when tool use requires a repeated procedure; otherwise the tool/MCP config itself is the artifact.
- **Agent** = acting specialist with potentially distinct model/tools/policies.
- **Subagent** = delegated temporary specialist spawned for a bounded task.
- **Evaluation** = project-specific quality test.
- **Benchmark** = external standardized comparison, not the main release gate.

That vocabulary minimizes collisions with current OpenAI/Codex usage while leaving room for project-specific planning. ŅłĆciteŅłéturn28view0Ņłéturn13view1Ņłéturn10view3Ņłéturn6view3Ņłéturn24search13Ņłéturn20view0Ņłéturn28view2Ņłéturn26search5Ņłü
